mysql基本操作指令

发表于 2018-06-22

mysql mysqldump 只导出表结构 不导出数据

mysqldump –opt -d 数据库名 -u root -p > xxx.sql

备份数据库

#mysqldump　数据库名　>数据库备份名

#mysqldump　-A　-u用户名　-p密码　数据库名>数据库备份名

#mysqldump　-d　-A　–add-drop-table　-uroot　-p　>xxx.sql

1.导出结构不导出数据

mysqldump　–opt　-d　数据库名　-u　root　-p　>　xxx.sql　　

2.导出数据不导出结构

mysqldump　-t　数据库名　-uroot　-p　>　xxx.sql　

3.导出数据和表结构

mysqldump　数据库名　-uroot　-p　>　xxx.sql　

4.导出特定表的结构

mysqldump　-uroot　-p　-B　数据库名　–table　表名　>　xxx.sql　　

导入数据：
　　由于mysqldump导出的是完整的SQL语句，所以用mysql客户程序很容易就能把数据导入了：
复制代码代码如下:

#mysql　数据库名　<　文件名

#source　/tmp/xxx.sql　　

以上操作如果出现长时间卡顿，请重启mysql服务解决问题。

protected void configure(HttpSecurity http) throws Exception{
   http

        //5 在/resources/static目录下的静态资源，Spring Security不拦截

web.ignoring().antMatchers("/resources/static/**");

    创建一个控制器，用于实现聊天室的业务，代码如下
    @Controller
public class ZskController {
@Autowired
private SimpMessagingTemplate simpMessagingTemplate;//通过该对象像浏览器发送消息
@MessageMapping("/chat")
public void  handleChat(Principal principal,String msg){//在mvc中，可以直接在参数中获得principal，包含当前用户的信息
System.out.println(principal.getName()+"------"+principal.toString());
System.out.println("--------------------");
if(principal.getName().equals("zsk")){//一个简单的判断，判断发送人来进行发送信息
    //通过convertAndSendToUser发送消息，第一个是接收消息的用户，第二个是浏览器的地址，第三个是消息本身。
    simpMessagingTemplate.convertAndSendToUser("gikky","/queue/notifications",principal.getName() +"-send:" + msg);
}else{
    simpMessagingTemplate.convertAndSendToUser("zsk","/queue/notifications",principal.getName() + "-send:"+msg);
    }
  }
}

Spring MVC常用注解

发表于 2018-05-26

Spring MVC的常用注解

  • @Controller

    @Controller注解在类上，表明这个类是Spring mvc的Controller，将其声明为Spring的一个Bean，Disoatcher Servlet会自动扫描注解了此注解的类，并将Web请求映射到注解@RequestMapping的方法上，这里需要注意的是,在声明普通的Bean的时候，使用@Component、@Server、@Repository和@Controller是等同的。因为@Service、@Repository、@Controller都组合了@Componment元注解。但是在Spring mvc声明控制器Bean的时候，只能使用@Controller。
  • @RequestMapping

    @RequestMapping注解是用来映射Web请求(访问路径和参数)、处理类和方法的。@RequestMapping可注解在类或方法上。注解在方法上的@RequestMapping路径会继承注解在类上的路径。
    @RequestMapping("/xxx")
public class aaa{
    @RequestMapping("/yyy")
    public void open(){
        }
    }
    以上代码中访问方法open的浏览器路径将是/xxx/yyy而不是/yyy，需要继承类上的路径。
    @RequestMapping支持Servlet的request和response作为参数，也支持对request和response的媒体类型进行配置
  • @ResponseBody

    @ResponseBody支持将返回值放在response体内，而非一个页面，我们在写基于Ajax的程序时，可以以此注解返回数据而非页面，此注解可放置在返回值前或者方法上。
  • @RequestBody

    @RequestBody允许request的参数在request体内，而不是在直接链接在地址后面，此注解放置在参数前。即：

    1、@RequestBody需要把所有请求参数作为json解析，因此，不能包含key=value这样的写法在请求url中，所有的请求参数都是一个json
    2、直接通过浏览器输入url时，@RequestBody获取不到json对象，需要用java编程或者基于ajax的方法请求，将Content-Type设置为application/json
  • @PathVariable

    @PathVariable用来接收路径参数，如/new/001,可接收001作为参数，此注解放置在参数前。
  • @RestController

    @RestController是一个组合注解，组合了@Controller和@RespoonseBody，这意味着当你只开发一个和页面交互数据的控制的时候，需要使用此注解，若没有此注解，要想实现上述功能，则需要自己在代码中加@Controller和@ResponseBody两个注解。

使用hexo

发表于 2018-05-23

如何部署hexo

hexo安装

安装git

hexo安装依赖于git和nodejs所以首先需要下载git和nodejs，git地址:https://git-for-windows.github.io/, mac用户可以使用brew直接install，安装完git以后输入指令git version查看到相应的版本就算安装完成。

安装NodeJs

hexo基于nodeJS环境的静态博客，且我们需要npm工具安装其他的包，所以需要进行安装，下载地址:
https://nodejs.org/en/ 下载msi文件以后双击直接安装，安装完成以后使用指令node -v查看版本号就算完成

安装hexo

安装hexo非常简单，先创建一个空文件夹，在命令行中进入该文件夹，使用npm i -g hexo 完成安装，npm是nodejs的包管理器，i指的是install -g是全局安装。安装完使用hexo -v可以查看相应的版本 。
安装完成以后使用hexo init即可生成一个hexo项目。

这是初始化后之前空文件夹的文件目录。

  • 说明一下

         node_modules:是依赖包

         public:存放的是生成的页面

         scaffolds:命令生成文章等的模板

         source:用命令创建的各种文章

         themes:主题

         _config.yml:整个博客的配置

         db.json:source解析所得到的

         package.json:项目所需要模块的配置信息

  • 配置到这里其实已经完成了本地的hexo部署了，使用hexo clean指令清除之前缓存，然后使用hexo g和hexo s生成项目文件和启动server，完成以后在浏览器中输入http://localhost:4000
    即可看到启动了的hexo应用了。

迁移到github

  • 创建一个github账号，并新建一个repository,名称为yourname.github.io 就是你的github用户名.github.io

  • 创建SSH

    在gitbash中输入: ssh-keygen -t rsa -C “youremail@example.com“，生成秘钥，并找到名为id_rsa.pub的文件，使用cat id_rsa.pub打开。

  • 将上面获取的ssh放到github中:


    添加一个new SSH key，title随便取，key就填写之前的id_rsa.pub中的内容。

最后操作

  • 用编辑器打开你的hexo项目文件夹目录下的_config.yml文件。将下面出现的节点修改成下面的样子(注意冒号后面的空格):
    deploy:
  type: git
  repo: https://github.com/YourgithubName/YourgithubName.github.io.git
  branch: master
  • 安装hexo-deployer-git包，用来将你的文章发布在github上。安装指令如下:
    npm install hexo-deployer-git --save
  • 执行命令(建议每次都按照如下步骤部署)
    hexo clean 
hexo generate
hexo deploy
    注意高版本的hexo已经支持简写 generate = g,deploy=d
    在deploy过程中可能需要输入你的username和password，这个是github的用户名及密码
  • 完成以后在浏览器中输入http://yourgithubname.github.io即可看到你的blog啦。
